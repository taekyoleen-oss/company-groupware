# 출근 체크 — 현장 셋업 & 운영 가이드

> Company Groupware 의 IP 기반 출근 체크 기능을 사무실에서 셋업하고 운영하기 위한 안내서입니다.
> GPS 기반 출근은 사용하지 않으며(코드는 `legacy/gps/` 에 보관), **사무실 IP + PC 등록 승인** 으로 일원화되었습니다.

---

## 1. 한눈에 보는 동작 원리

```
[직원 PC] ─(회사 WiFi/유선)→ [회사 라우터] ─(인터넷)→ [그룹웨어 서버]
                                  │
                                  └─ 외부에서 보이는 공인 IP : 203.0.113.45
                                          ▲
                                          └─ 이 IP 가 cg_office_networks 에 등록되어 있어야 출근 체크 가능
```

| 판정 단계 | 검사 항목 | 실패 시 |
|---|---|---|
| 1) 네트워크 | 클라이언트 공인 IP 가 등록된 사무실 IP/CIDR 안에 들어오는가 | "사무실 네트워크가 아닙니다" → 출근 버튼 비활성 |
| 2) PC 등록 (선택) | 승인 필수 모드 ON 시, 본인의 브라우저(User-Agent) 가 `approved` 상태인가 | "이 PC 는 아직 관리자 승인 대기 중입니다" → 버튼 비활성 |
| 3) 중복 확인 | 오늘 날짜로 이미 출근 기록이 있는가 | "이미 출근 확인이 완료되었습니다" (409) |

## 2. 기본 셋업 (앱관리자, 최초 1회)

### 2-1. DB 마이그레이션 실행
Supabase Dashboard → **SQL Editor** 에서 아래 두 파일을 차례로 Run.

| 순서 | 파일 | 작업 |
|---|---|---|
| ① | `output/step13_office_ip_unification.sql` | `cg_office_networks` 테이블 + RLS |
| ② | `output/step14_office_devices.sql` | `cg_office_devices` 테이블 + `cg_company_settings.require_device_approval` 컬럼 |

> 두 SQL 모두 멱등(再실행 안전) 입니다. 이미 적용된 경우에도 다시 실행해 됩니다.

### 2-2. 사무실 PC 에서 회사 IP 등록
**반드시 사무실 인터넷에 연결된 상태**로 다음 절차를 따릅니다.

1. 앱관리자 계정으로 로그인 → `/admin` 진입
2. **회사 설정** 탭 → "허용 IP 주소" 카드
3. 두 가지 등록 방법
   - **자동**: `현재 내 IP 자동 등록` 클릭 → 현재 공인 IP 가 `/32` 로 추가됨
   - **수동**: CIDR 직접 입력 (예: `211.219.53.239` 또는 `211.219.53.0/24`)
4. 라벨(예: `본사`, `부설연구소`)을 입력 후 저장

> **💡 라벨 팁**: 여러 회선이 있을 때 어느 회선의 IP 인지 구분하기 쉽도록 사무실 위치 + 회선 종류를 적으세요. (예: `본사-KT`, `2층-SK`)

### 2-3. PC 승인 정책 결정
회사 설정 탭 최상단의 **PC 승인 정책** 카드에서:

| 옵션 | 동작 |
|---|---|
| ☐ OFF (기본값) | 사무실 IP 면 누구나 출근 체크 가능. PC 등록은 "어떤 PC 가 쓰이는지 기록"하는 용도로만 사용 |
| ☑ ON (승인 필수) | 사무실 IP + 관리자가 승인한 PC 에서만 출근 체크 가능. 보안이 중요한 환경에 권장 |

## 3. 직원 사용 흐름

### 3-1. 처음 출근하는 직원
1. 사무실에서 그룹웨어 로그인 → 프로필 → **출근** 탭
2. "사무실 네트워크에 연결되어 있습니다" 초록 표시 확인
3. (선택) **PC 등록 요청** 버튼 → 라벨 입력 ("내 자리 PC" 등) → 관리자 승인 대기
4. 출근 확인 버튼 클릭

### 3-2. 승인 필수 모드 일 때
- 미등록/미승인 PC: 버튼이 비활성화되며 "관리자 승인된 PC 에서만 출근 체크가 가능합니다" 메시지 표시
- 승인 대기 중: "승인 대기 중" 노란 배지 표시
- 거절된 경우: "다시 요청" 버튼이 보임

### 3-3. 다른 PC 사용 (회의실 PC 등)
- 다른 PC 의 브라우저는 별도 등록이 필요합니다 (User-Agent 가 다르기 때문)
- 같은 PC 라도 Chrome 과 Edge 를 번갈아 쓰면 각각 등록되어야 함 → 한 가지 브라우저로 통일 권장

---

## 4. 관리자 운영 가이드

### 4-1. PC 등록 요청 처리
`/admin` → 회사 설정 탭 → **등록된 PC** 섹션:
- 상태 배지: 🟡 대기 / 🟢 승인 / 🔴 거절
- 액션: `✓ 승인` / `🛡 거절` / `🗑 삭제`
- 사용자별 아바타·이름·User-Agent·마지막 사용 IP 노출
- 라벨은 관리자도 자유롭게 편집 가능 (예: 사용자가 입력한 "내 컴퓨터" → 관리자가 "총무팀 회의실 PC" 로 보정)

### 4-2. 부서 이동·퇴사 시 정리
- 퇴사자: 등록된 PC 삭제 권장
- 부서/자리 이동: 라벨만 갱신해도 충분 (User-Agent 유지)

## 5. 유동 IP 환경 (대부분의 일반 인터넷 회선)

대부분의 ISP(KT/SK/LG U+)는 사업자용 회선이 아니면 IP 가 가끔 바뀝니다. 다음 패턴을 적용하세요.

### 5-1. 같은 ISP 의 같은 지역대(/24) 단위로 등록

```
- 사용자가 받는 IP : 211.219.53.45 → 다음 달 211.219.53.198 로 바뀜
- 두 IP 모두 같은 /24 안에 들어옴 → 211.219.53.0/24 로 등록하면 OK
```

**확인 방법**
1. 사무실 PC 에서 `https://whatismyipaddress.com` 또는 그룹웨어의 `현재 내 IP 자동 등록` 으로 IP 확인
2. 1–2주 사이 두세 번 다시 확인
3. 첫 세 옥텟(예: `211.219.53.`) 까지 같으면 `/24` 로 등록하면 안정

### 5-2. ISP 의 IP 풀이 더 넓은 경우 (/16)
- 일부 ISP 는 더 넓은 대역에서 IP 를 발급 → 그러나 `/16` 은 65,536 개 IP 가 허용되어 보안상 위험
- 대안 1: 회사 회선을 **고정 IP** 로 변경 신청 (월 1~2만원 추가, 가장 권장)
- 대안 2: 승인 필수 모드 ON + `/24` 등록 → IP 가 바뀌어도 PC 가 승인되어 있으면 큰 문제 없음

### 5-3. 고정 IP 신청 (가장 깔끔)
- ISP 영업 / 고객센터 → "기업용 고정 IP 회선" 요청
- 비용: 기본 회선 + 고정 IP 옵션 월 1~3만원
- 한 번 고정되면 `xxx.xxx.xxx.xxx/32` 로 등록 → 영구 안정

## 6. 여러 회선 / 여러 사무실 운영

### 6-1. 같은 건물에 회선 2개
예: 본사가 KT(메인) + SK(백업) 회선을 모두 운영 중

```
사무실 → 라우터 ┬→ KT 회선   → 공인 IP 211.219.53.10
                └→ SK 회선   → 공인 IP 175.123.45.67
```

**등록 방법**: 두 IP 를 모두 등록 (라벨 각각 `본사-KT`, `본사-SK`)

### 6-2. 분점/지점 운영
지점별로 회선이 다르므로 각 지점 IP 를 따로 등록:

| 라벨 | CIDR | 비고 |
|---|---|---|
| 본사 | 211.219.53.10/32 | KT 고정 IP |
| 강남지점 | 175.123.45.0/24 | SK 동적 (지역대 등록) |
| 부설연구소 | 218.234.7.55/32 | LG U+ 고정 |

각 지점의 직원은 자기 지점 사무실에서 출근 체크 → 알맞은 IP 가 매칭되어 동작

### 6-3. 외근 / 재택 인원이 있는 경우
**현재 시스템은 외근/재택 출근을 지원하지 않습니다.**

해결 옵션:
- (간이) 외근/재택일 별도 휴가 카테고리 사용 → 출근 체크 없이 일정으로만 기록
- (정식) 회사 VPN 도입 → VPN 출구 IP 를 사무실 IP 와 동일하게 → VPN 접속한 재택 직원도 사무실 IP 로 인식됨
- (재도입) `legacy/gps/` 의 GPS 출근 기능을 부활시켜 외근지에서 위치 기반 출근 허용

## 7. 트러블슈팅

| 증상 | 원인 | 대응 |
|---|---|---|
| **"현재 IP 가 사무실 IP 가 아닙니다"** (사내에서) | IP 가 바뀌었거나 다른 회선으로 잡힘 | 관리자에게 IP 확인 요청 → 새 IP 추가 또는 `/24` 로 등록 변경 |
| **"이 PC 는 아직 관리자 승인 대기 중입니다"** | 승인 필수 모드 ON, 디바이스 pending | 관리자가 `/admin` → 등록된 PC 에서 승인 |
| **모바일 LTE 로 접속해도 출근 안 됨** | 통신사 IP 는 사무실 IP 와 다름 | 정상 동작 (의도된 차단). 사무실 WiFi 에 붙어서 시도 |
| **두 사람이 같은 PC 로 출근 체크** | User-Agent 가 같아 같은 디바이스로 인식되지만 user_id 가 달라 각각 별도 등록됨 | 정상. 둘 다 따로 승인 받으면 됨 |
| **앱관리자가 IP 등록을 누르니 에러** | `output/step13_office_ip_unification.sql` 미적용 | 위 2-1 절차 다시 실행 |
| **회사가 VPN 사용 — 어느 IP 가 잡힘?** | VPN 출구 IP 가 잡힘 | VPN 출구 IP 를 사무실 IP 로 등록하면 VPN 사용자도 출근 가능 (단, 보안 정책 검토 필요) |

## 9. 참고 — 데이터 모델

| 테이블 | 역할 |
|---|---|
| `cg_company_settings.require_device_approval` | 승인 필수 모드 토글 |
| `cg_office_networks` | 허용 IP/CIDR 목록 |
| `cg_office_devices` | PC 등록 요청 + 승인 상태 (user_id × user_agent 유니크) |
| `cg_attendance` | 일자별 출근 기록 (KST 기준 `date`) |

| API 경로 | 메서드 | 설명 |
|---|---|---|
| `/api/attendance/ip-check` | GET | 현재 IP 매칭 + 디바이스 상태 + 회사 정책 조회 |
| `/api/attendance` | POST | 출근 기록 생성 (IP + 승인 가드) |
| `/api/attendance/device-register` | POST | 직원 PC 등록 요청 |
| `/api/attendance/device-status` | GET | 본인 디바이스 상태 조회 |
| `/api/admin/office-devices` | GET | 전체 디바이스 목록 (관리자) |
| `/api/admin/office-devices/[id]` | PATCH/DELETE | 승인·거절·라벨·삭제 (관리자) |
| `/api/admin/office-networks` | GET/POST | IP/CIDR 목록·추가 |
| `/api/admin/office-networks/[id]` | PATCH/DELETE | IP 라벨 편집·삭제 |
